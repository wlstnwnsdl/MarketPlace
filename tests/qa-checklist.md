# QA 체크리스트

> **관리 규칙**: 기능 추가·수정·삭제 시 이 파일을 반드시 갱신한다.
> - 새 기능 추가 → 관련 카테고리에 항목 추가
> - 동작 변경 → 해당 항목 수정
> - 기능 제거 → 해당 항목 삭제 (삭제 이유를 커밋 메시지에 기록)
>
> 각 항목 뒤 `[기능태그]`는 어떤 기능에 속하는지 나타낸다. 기능 변경 시 태그로 관련 항목을 검색한다.

---

## UI/UX

### 접근 제어 (프론트엔드)
- [ ] 비로그인 상태에서 홈 페이지 접근 가능한가? `[인증]`
- [ ] 비로그인 상태에서 프롬프트 상세 페이지 접근 가능한가? `[인증]`
- [ ] 비로그인 상태에서 "구매하기" 클릭 시 로그인 페이지로 이동하는가? `[인증][구매]`
- [ ] `/upload` 비로그인 접근 시 로그인 페이지로 리다이렉트되는가? `[인증][프롬프트등록]`
- [ ] `/mypage` 비로그인 접근 시 로그인 페이지로 리다이렉트되는가? `[인증][마이페이지]`

### 프롬프트 등록/수정
- [ ] content 50KB 초과 입력 시 업로드 버튼이 비활성화되는가? `[프롬프트등록]`
- [ ] content 50KB 초과 시 경고 메시지가 표시되는가? `[프롬프트등록]`
- [ ] 바이트 사용량 프로그레스바가 실시간으로 갱신되는가? `[프롬프트등록]`
- [ ] 수정 모드에서 기존 데이터가 폼에 올바르게 로드되는가? `[프롬프트수정]`
- [ ] 수정 모드에서 타입(type) 필드가 변경 불가 상태로 표시되는가? `[프롬프트수정]`

### 프롬프트 상세
- [ ] 자신의 프롬프트 상세에서 "구매하기" 대신 "수정하기" 버튼이 표시되는가? `[프롬프트상세][소유자]`
- [ ] 구매 전에는 previewContent만 표시되는가? `[프롬프트상세][접근제어]`
- [ ] 구매 후 전체 content가 표시되는가? `[프롬프트상세][접근제어][구매]`
- [ ] 다운로드 후 Blob URL이 정리(revokeObjectURL)되는가? `[다운로드]`

### 헤더/내비게이션
- [ ] 로그인 후 헤더에 사용자 이름 첫 글자가 표시되는가? `[인증][프로필]`
- [ ] 프로필 버튼 클릭 시 이메일/이름이 드롭다운으로 표시되는가? `[인증][프로필]`
- [ ] 드롭다운 외부 클릭 시 닫히는가? `[인증][프로필]`

### 마이페이지
- [ ] 구매한 프롬프트 탭과 내가 등록한 탭이 정상 동작하는가? `[마이페이지]`
- [ ] 내가 등록한 프롬프트에 상태(공개/미공개/대기) 배지가 표시되는가? `[마이페이지][프롬프트상태]`
- [ ] 삭제 확인 다이얼로그가 표시되는가? `[마이페이지][프롬프트삭제]`

## 보안

### 인증 경계
- [ ] 만료된 accessToken으로 요청 시 자동 재발급 후 원본 요청이 재시도되는가? `[인증][토큰]`
- [ ] refreshToken도 만료 시 로그인 페이지로 리다이렉트되는가? `[인증][토큰]`

### 권한 검사 (API)
- [ ] 미구매자가 `GET /api/prompts/{id}/download` 호출 시 403 응답하는가? `[다운로드][접근제어]`
- [ ] 타인 프롬프트 `DELETE /api/prompts/{id}` 시도 시 403 응답하는가? `[프롬프트삭제][접근제어]`
- [ ] 타인 프롬프트 `PUT /api/prompts/{id}` 시도 시 403 응답하는가? `[프롬프트수정][접근제어]`
- [ ] 자기 프롬프트 구매 시도 시 400 응답하는가? `[구매][접근제어]`
- [ ] PENDING/PRIVATE 프롬프트를 타인이 조회 시 404 응답하는가? `[프롬프트상태][접근제어]`

## 데이터 무결성

- [ ] content 없이 프롬프트 등록 시 400 응답하는가? `[프롬프트등록][유효성검사]`

### 콘텐츠 접근

- [ ] 구매자는 판매자가 PRIVATE으로 변경해도 다운로드 가능한가? `[다운로드][프롬프트상태]`
