# 01. 공유 컨텍스트 분리 및 테스트 파일 관리 체계 구축

**날짜**: 2026-04-22
**작업 유형**: chore
**커밋**: `ff3aaa6`

## 작업 내용

에이전트 파일(architect, developer, qa, user)마다 중복으로 작성된 프로젝트 컨텍스트와
공통 절차를 공유 파일로 분리했다. 동시에 QA/라이브 테스트 항목을 에이전트 파일 안에
인라인으로 두지 않고 `tests/` 폴더의 독립 파일로 분리하여, 기능 변경 시 테스트 목록을
별도로 관리할 수 있게 했다.

## 변경 파일

### 신규 생성
- `.claude/agents/CLAUDE.md` — 모든 에이전트 공통 프로젝트 컨텍스트 (스택, 도메인 모델, CRITICAL 규칙, API 목록, 환경 변수)
- `.claude/agents/SKILL.md` — 공통 절차 (빌드/테스트 명령, 서버 라이프사이클, 작업 로그 규칙, 커밋 컨벤션, 코딩 컨벤션)
- `tests/qa-checklist.md` — UI/UX·보안·데이터 무결성 QA 체크리스트 36개 항목, `[기능태그]` 부여
- `tests/live-scenarios.md` — API 라이브 테스트 시나리오 22개, 그룹(A~F) 분류, `[기능태그]` 부여

### 간소화 (중복 내용 제거)
- `.claude/agents/architect.md` — 73줄 → 27줄 (프로젝트 컨텍스트·도메인·서버 명령어 제거)
- `.claude/agents/developer.md` — 163줄 → 42줄 (패키지 구조·코딩 컨벤션·작업 로그 규칙 제거) + 테스트 파일 갱신 규칙 추가
- `.claude/agents/qa.md` — 137줄 → 41줄 (체크리스트 외부화, `tests/qa-checklist.md` 읽도록 변경)
- `.claude/agents/user.md` — 247줄 → 70줄 (시나리오 외부화, `tests/live-scenarios.md` 읽도록 변경)
- `CLAUDE.md` — 에이전트 컨텍스트 파일 및 `tests/` 폴더 참조 섹션 추가

## 핵심 결정 / 주의사항

- 테스트 파일의 `[기능태그]` 체계: 기능 변경 시 태그로 관련 항목을 검색해 갱신한다.
  예) 구매 기능 변경 → `[구매]` 태그 항목 전체 검토
- `developer.md` 작업 절차 6번에 "테스트 파일 갱신"이 추가되어 구현 완료 후 강제된다.
- qa/user 에이전트는 시작 시 순서대로 CLAUDE.md → SKILL.md → tests/*.md 를 읽는다.
