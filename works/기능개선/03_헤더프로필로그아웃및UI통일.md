# 03. 헤더 프로필 로그아웃 및 UI 통일

**날짜**: 2026-04-23
**작업 유형**: feat / refactor

## 작업 내용

name이 null인 기존 사용자가 헤더 드롭다운에서 항상 "사용자"로 표시되는 문제를 수정했다.
아바타 이니셜과 드롭다운 표시를 name 우선 → email 폴백 방식으로 변경하고,
로그아웃 기능(토큰 삭제 후 홈 이동)을 드롭다운에 추가했다.
mp-btn-terminal CSS 클래스의 text-xs를 text-sm으로 통일해 [수정하기] 버튼과 폰트 크기를 맞췄다.
PromptCard의 기본 버튼 라벨을 CLI 명령어 형식에서 "상세보기"로 변경했다.

## 변경 파일

- `frontend/src/components/Header.tsx` — 아바타 이니셜 로직, 드롭다운 내용(name/email 폴백), 로그아웃 버튼 추가
- `frontend/src/index.css` — mp-btn-terminal: text-xs → text-sm
- `frontend/src/components/PromptCard.tsx` — 기본 버튼 라벨: `> marketplace get {id}` → `상세보기`
- `tests/qa-checklist.md` — 헤더/내비게이션 항목 갱신 (name 폴백, 로그아웃 항목 추가)

## 핵심 결정 / 주의사항

- 드롭다운에서 name이 없을 때 email 한 줄만 표시 (email이 곧 식별자 역할)
- name이 있을 때만 email을 두 번째 줄에 추가 표시
- 로그아웃은 localStorage 직접 제거 후 `window.location.href = '/'` 로 전체 상태 초기화
- PromptDetailPage.tsx의 다운로드/구매 버튼은 mp-btn-terminal 클래스를 사용하므로 CSS 변경으로 자동 반영됨 (별도 수정 불필요)
