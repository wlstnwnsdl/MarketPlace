# 01. 7개 기능 일괄 구현

**날짜**: 2026-04-22  
**작업 유형**: feat

## 작업 내용

오케스트레이터 지시에 따라 7개 항목을 순차적으로 구현했다. 각 항목은 독립적이지만 PromptStatus enum 신설이 백엔드 전반(entity, DTO, repository, service, controller)과 프론트엔드(types, api, pages) 모두에 영향을 미쳐 변경 범위가 컸다. PromptCard.tsx의 ROLE_LABEL 누락으로 첫 빌드가 실패했고 즉시 수정해 재빌드 성공했다.

## 핵심 결정 / 주의사항

- `findPrompts` (홈 목록)는 항상 `status = PUBLIC`으로 필터링 — PENDING/PRIVATE 프롬프트는 홈에 노출되지 않음
- PUBLIC이 아닌 프롬프트 상세 조회 시 소유자가 아니면 PromptNotFoundException(404) 반환
- 태그 검색은 LEFT JOIN + DISTINCT 대신 EXISTS 서브쿼리 사용 — 페이지네이션 count 오류 방지
- updatePrompt API payload에 type 필드가 누락되어 있었으므로 추가 (서버는 기존 type 유지하지만 validation상 @NotNull이므로 전달 필요)
- PromptCard의 ROLE_LABEL도 TargetRole Record이므로 새 역할 추가 시 반드시 함께 수정해야 함 (첫 빌드에서 발견)
